We need to fix two backend bugs in server/src/modules/notifications/app-events.service.ts that are causing queued SSE notifications to be dropped.

1. NestJS Fast-Emit Drop: drainPending resolves so fast that RxJS emits the queued notifications before the client's EventSource connection is fully established. We must add a delay to the pending$ observable.
2. Connection Tracking Bug: connectedUsers is a Set. Because the frontend opens two SSE connections, if one connection briefly drops, disconnect deletes the user from the Set completely, breaking the online/offline routing logic. It needs to be a Map that tracks the connection count.

Please make the following exact updates to server/src/modules/notifications/app-events.service.ts:
